Actionable Setup Guide: Claude Code Memory ManagementBased on the research regarding tiered memory and context saturation, here is how to practically apply these concepts to your local Claude Code setup to prevent the agent from "forgetting" early decisions over long coding sessions.1. Project Initialization (Progressive Disclosure)Do not let the agent blindly read your entire repository into its context window on startup.Action: Create two static files at the root of your project:project_manifest.md or ARCHITECTURE.md: A high-level index of where things live and the core tech stack.last_session_summary.md: A running log of where you left off.Startup Habit: When starting Claude Code, begin with a prompt like: "Read project_manifest.md and last_session_summary.md to get context, but do not read the full source files yet."2. Customize the Compaction Prompt ("Smart Pivot")Standard auto-summaries often drop the reasons behind code changes, logging only the actions.Action: If you use a custom system prompt or a wrapper script for Claude Code, inject instructions requiring the "Smart Pivot" format for all summaries. Whenever you ask Claude to summarize its work, force it to use this exact structure:STATUS: (e.g., Fix in progress)CONTEXT: (The "Why" - e.g., Schema migration caused regressions)DECISION: (e.g., Made user object optional)ACTION: (e.g., Updated auth.ts)NEXT: (Immediate next step)3. Implement a "Manual Memory" Workflow (Hot to Cold Tier)Don't rely purely on Claude's internal context window (Hot Tier) or background auto-compaction (Warm Tier) to remember critical architectural shifts.Action: Create a DECISIONS.md file (Cold Tier).Habit: Treat this as a manual memory trigger. Whenever you and Claude solve a hard bug, change a database schema, or switch libraries, explicitly command: "Document why we just abandoned the WebSocket approach and chose long-polling in DECISIONS.md." ## 4. Session Teardown (Warm to Cold Tier)Clear out the context window effectively before you finish for the day to ensure tomorrow's session starts fresh but informed.Action: Before closing Claude Code, prompt it to perform a session audit: "Summarize today's work, the current state of the project, and unresolved blockers into last_session_summary.md using the Smart Pivot format."Advanced: If you use a tool like Specstory or Cursor, use their timeline/diff features to generate a narrative diff of the session and paste that into your summary document.
