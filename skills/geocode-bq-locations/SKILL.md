# Geocode BQ Locations

Fills in missing or incorrect coordinates for stores in `normalized_klever_stores_mapping`. Generates a Python script that runs locally, respects Nominatim's rate limits, and produces a CSV with QA flags ready for BQ upload or manual review.

Per the CLAUDE.md data rule: data stays on disk. This skill generates a self-contained script the user runs locally. Results are written to a CSV, not loaded into the conversation.

---

## When to Use

- Onboarding a new advertiser to the proximity map and their stores have NULL, zero, or clearly-wrong coordinates
- User says "geocode stores for X", "add coordinates for X", "add lat/lon to stores", "geocode for Placer", or "onboard X locations"
- After running `klever-bq-store-lookup` and the QA flags report coordinate issues

---

## Prerequisites

Read `klever-bq-store-lookup` skill before starting. Use it to:
1. Resolve the DSP string ID for the advertiser
2. Pull the current store list and surface coordinate QA flags

---

## Workflow

### Step 1 — Identify Stores to Geocode

Run `klever-bq-store-lookup` for the advertiser. Stores requiring geocoding are those where:
- `LATITUDE IS NULL OR LONGITUDE IS NULL`
- `LATITUDE = 0 OR LONGITUDE = 0`
- Coordinate bounds fail (LATITUDE outside 24.0--50.0, LONGITUDE outside -125.0 to -66.0 for US stores)

Export the target rows to a local CSV:

```bash
bq query --use_legacy_sql=false --format=csv \
  "SELECT KLEVER_LOCATION_ID, LOCATION_NAME, ADDRESS, CITY, STATE, ZIP_CODE, LATITUDE, LONGITUDE
   FROM \`prj-p-biz-report-fo53kywlio.klever_external_data.normalized_klever_stores_mapping\`
   WHERE ADVERTISER_ID = '<dsp_string_id>'
     AND INACTIVE = FALSE
     AND (LATITUDE IS NULL OR LONGITUDE IS NULL
          OR LATITUDE = 0 OR LONGITUDE = 0
          OR LATITUDE NOT BETWEEN 24.0 AND 50.0
          OR LONGITUDE NOT BETWEEN -125.0 AND -66.0)
   ORDER BY STATE, CITY" \
  > /tmp/stores-to-geocode-<advertiser>-$(date +%Y%m%d).csv
```

Tell the user the count before proceeding. If the count is 0, there is nothing to do.

### Step 2 — Generate the Geocoding Script

Write the script to the ticket folder (or `/tmp/` if no ticket context). Name it `geocode_stores_<advertiser>_<YYYYMMDD>.py`.

The script must:
- Read the CSV exported in Step 1
- Geocode each row via Nominatim using a progressive address simplification strategy
- Respect the 1 req/sec rate limit (hard `time.sleep(1)` between every request, no batching)
- Write results to a new CSV with all original columns plus: `GEOCODED_LAT`, `GEOCODED_LON`, `GEO_CONFIDENCE`, `GEO_MATCH_TYPE`, `GEO_QA_FLAG`
- Resume from partial results if the output CSV already exists (skip rows already processed)
- Print progress to stdout with `flush=True`
- Use only Python standard library (no pip dependencies)

#### Script Template

```python
#!/usr/bin/env python3
"""
Geocode stores for <ADVERTISER> from normalized_klever_stores_mapping.
Generated by geocode-bq-locations skill.

Usage:
    python geocode_stores_<advertiser>_<date>.py <input_csv> [output_csv]

Nominatim rate limit: 1 req/sec (enforced with time.sleep(1)).
No external dependencies required (stdlib only).
"""

import csv
import json
import os
import re
import sys
import time
from urllib.parse import urlencode
from urllib.request import urlopen, Request

NOMINATIM_URL = "https://nominatim.openstreetmap.org/search"
USER_AGENT = "KleverGeocodingScript/1.0 (gamyot@beklever.com)"

# QA thresholds
US_LAT_MIN, US_LAT_MAX = 24.0, 50.0
US_LON_MIN, US_LON_MAX = -125.0, -66.0
HIGH_CONFIDENCE_TYPES = {"house", "building", "amenity", "shop"}
MEDIUM_CONFIDENCE_TYPES = {"commercial", "retail"}
LOW_CONFIDENCE_TYPES = {"postcode", "suburb", "city", "town", "county", "state"}


def nominatim_geocode(query: str) -> dict | None:
    """Query Nominatim. Returns first result or None."""
    params = {
        "q": query,
        "format": "json",
        "addressdetails": 1,
        "limit": 1,
        "countrycodes": "us",
    }
    url = f"{NOMINATIM_URL}?{urlencode(params)}"
    req = Request(url, headers={"User-Agent": USER_AGENT})
    try:
        with urlopen(req, timeout=10) as resp:
            results = json.loads(resp.read())
            return results[0] if results else None
    except Exception as e:
        print(f"  [WARN] Nominatim error for '{query}': {e}", flush=True)
        return None


def build_address_attempts(row: dict) -> list[str]:
    """Return progressively simplified address attempts."""
    address = row.get("ADDRESS", "").strip()
    city = row.get("CITY", "").strip()
    state = row.get("STATE", "").strip()
    zip_code = row.get("ZIP_CODE", "").strip()

    attempts = []

    # 1. Full address with zip
    if address and city and state and zip_code:
        attempts.append(f"{address}, {city}, {state} {zip_code}")

    # 2. Full address without zip
    if address and city and state:
        attempts.append(f"{address}, {city}, {state}")

    # 3. Strip suite/unit/floor from address
    stripped = re.sub(
        r"(?i)\s*(suite|ste|unit|#|apt|floor|fl|bldg|building)\s*[\w-]+", "", address
    ).strip().rstrip(",")
    if stripped != address and stripped and city and state:
        attempts.append(f"{stripped}, {city}, {state}")

    # 4. City + state + zip (centroid fallback)
    if city and state and zip_code:
        attempts.append(f"{city}, {state} {zip_code}")

    # 5. City + state only
    if city and state:
        attempts.append(f"{city}, {state}")

    return attempts


def classify_result(result: dict) -> tuple[str, str]:
    """
    Returns (confidence, match_type).
    confidence: HIGH / MEDIUM / LOW / CENTROID
    match_type: the OSM class/type string
    """
    osm_type = result.get("type", "unknown")
    osm_class = result.get("class", "unknown")
    category = f"{osm_class}/{osm_type}"

    if osm_type in HIGH_CONFIDENCE_TYPES or osm_class in ("building",):
        return "HIGH", category
    if osm_type in MEDIUM_CONFIDENCE_TYPES or osm_class in ("landuse",):
        return "MEDIUM", category
    if osm_type in LOW_CONFIDENCE_TYPES or osm_class in ("place", "boundary"):
        return "CENTROID", category
    return "LOW", category


def qa_flag(row: dict, lat: float, lon: float, confidence: str, attempt_index: int) -> str:
    """Return a pipe-separated list of QA flags, or empty string if clean."""
    flags = []

    # Wrong country (outside continental US bounding box)
    if not (US_LAT_MIN <= lat <= US_LAT_MAX and US_LON_MIN <= lon <= US_LON_MAX):
        flags.append("WRONG_COUNTRY_OR_REGION")

    # Centroid-only match
    if confidence == "CENTROID":
        flags.append("CENTROID_ONLY")

    # Low confidence
    if confidence == "LOW":
        flags.append("LOW_CONFIDENCE")

    # Fell back past full address
    if attempt_index >= 3:
        flags.append(f"SIMPLIFIED_FALLBACK_L{attempt_index + 1}")

    return "|".join(flags)


def geocode_row(row: dict, row_num: int, total: int) -> dict:
    """Geocode a single store row. Returns enriched row dict."""
    attempts = build_address_attempts(row)
    location_id = row.get("KLEVER_LOCATION_ID", "?")
    print(
        f"  [{row_num}/{total}] {location_id} -- {attempts[0] if attempts else 'NO ADDRESS'}",
        flush=True,
    )

    for i, attempt in enumerate(attempts):
        time.sleep(1)  # Nominatim rate limit: 1 req/sec
        result = nominatim_geocode(attempt)
        if result:
            lat = float(result["lat"])
            lon = float(result["lon"])
            confidence, match_type = classify_result(result)
            flag = qa_flag(row, lat, lon, confidence, i)
            if flag:
                print(f"    [QA] {flag}", flush=True)
            return {
                **row,
                "GEOCODED_LAT": lat,
                "GEOCODED_LON": lon,
                "GEO_CONFIDENCE": confidence,
                "GEO_MATCH_TYPE": match_type,
                "GEO_QA_FLAG": flag,
            }

    print(f"    [FAIL] No result for any attempt", flush=True)
    return {
        **row,
        "GEOCODED_LAT": "",
        "GEOCODED_LON": "",
        "GEO_CONFIDENCE": "FAILED",
        "GEO_MATCH_TYPE": "",
        "GEO_QA_FLAG": "GEOCODE_FAILED",
    }


def load_already_processed(output_path: str) -> set[str]:
    """Return set of KLEVER_LOCATION_IDs already in the output CSV."""
    if not os.path.exists(output_path):
        return set()
    processed = set()
    with open(output_path, newline="") as f:
        reader = csv.DictReader(f)
        for row in reader:
            lid = row.get("KLEVER_LOCATION_ID", "")
            if lid:
                processed.add(lid)
    print(f"[RESUME] {len(processed)} rows already processed, skipping.", flush=True)
    return processed


def main():
    if len(sys.argv) < 2:
        print("Usage: python <script>.py <input_csv> [output_csv]")
        sys.exit(1)

    input_path = sys.argv[1]
    output_path = sys.argv[2] if len(sys.argv) > 2 else input_path.replace(".csv", "_geocoded.csv")

    print(f"Input:  {input_path}", flush=True)
    print(f"Output: {output_path}", flush=True)

    with open(input_path, newline="") as f:
        reader = csv.DictReader(f)
        rows = list(reader)

    total = len(rows)
    print(f"Total rows to geocode: {total}", flush=True)

    already_processed = load_already_processed(output_path)

    output_fieldnames = list(rows[0].keys()) + [
        "GEOCODED_LAT", "GEOCODED_LON", "GEO_CONFIDENCE", "GEO_MATCH_TYPE", "GEO_QA_FLAG"
    ]

    write_header = not os.path.exists(output_path)
    results = []
    failed = []

    with open(output_path, "a", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=output_fieldnames)
        if write_header:
            writer.writeheader()

        for i, row in enumerate(rows, 1):
            lid = row.get("KLEVER_LOCATION_ID", "")
            if lid in already_processed:
                continue
            enriched = geocode_row(row, i, total)
            writer.writerow(enriched)
            f.flush()
            results.append(enriched)
            if enriched["GEO_CONFIDENCE"] == "FAILED":
                failed.append(enriched)

    # Summary
    processed_count = len(results)
    flagged = [r for r in results if r.get("GEO_QA_FLAG")]
    print(f"\n=== Summary ===", flush=True)
    print(f"Processed:  {processed_count}", flush=True)
    print(f"Failed:     {len(failed)}", flush=True)
    print(f"QA flagged: {len(flagged)}", flush=True)
    if flagged:
        print(f"\nQA-flagged rows:", flush=True)
        for r in flagged:
            print(
                f"  {r['KLEVER_LOCATION_ID']} | {r['GEO_QA_FLAG']} | "
                f"{r.get('ADDRESS')}, {r.get('CITY')}, {r.get('STATE')}",
                flush=True,
            )
    print(f"\nOutput written to: {output_path}", flush=True)


if __name__ == "__main__":
    main()
```

### Step 3 — Run the Script

Tell the user to run:

```bash
python geocode_stores_<advertiser>_<date>.py /tmp/stores-to-geocode-<advertiser>-<date>.csv
```

For large batches (200+ stores), suggest running in the background:

```bash
nohup python geocode_stores_<advertiser>_<date>.py /tmp/stores-to-geocode-<advertiser>-<date>.csv \
  > /tmp/geocode-<advertiser>-$(date +%Y%m%d).log 2>&1 &
echo "PID: $!"
```

The script resumes automatically if interrupted (skips already-written rows on restart).

### Step 4 — Review QA Flags

After the script completes, review the flagged rows in the output CSV:

| Flag | Meaning | Action |
|---|---|---|
| `WRONG_COUNTRY_OR_REGION` | Coordinates fall outside continental US bounds | Manual review. Likely an address parsing issue |
| `CENTROID_ONLY` | Matched to city/county centroid, not street address | Acceptable for city-level pins. Flag for Placer if street precision needed |
| `LOW_CONFIDENCE` | Matched to an ambiguous OSM category | Verify against Google Maps before writing to BQ |
| `SIMPLIFIED_FALLBACK_L3+` | Needed aggressive simplification to get a hit | Review the original address for typos |
| `GEOCODE_FAILED` | No match on any attempt | Manual lookup required |

Present the QA summary to the user before proceeding to Step 5.

### Step 5 — Write Results Back

#### Option A: BQ UPDATE (preferred for < 500 rows)

Generate UPDATE statements from the output CSV. Only update rows with HIGH or MEDIUM confidence and no QA flags:

```python
import csv

with open("output_geocoded.csv") as f:
    for row in csv.DictReader(f):
        if row["GEO_CONFIDENCE"] in ("HIGH", "MEDIUM") and not row["GEO_QA_FLAG"]:
            lid = row["KLEVER_LOCATION_ID"]
            lat = row["GEOCODED_LAT"]
            lon = row["GEOCODED_LON"]
            print(
                f"UPDATE `prj-p-biz-report-fo53kywlio.klever_external_data.normalized_klever_stores_mapping` "
                f"SET LATITUDE = {lat}, LONGITUDE = {lon} "
                f"WHERE KLEVER_LOCATION_ID = '{lid}';"
            )
```

**Do NOT update rows with `WRONG_COUNTRY_OR_REGION` or `GEOCODE_FAILED` flags without manual review.**

Before running any UPDATE: follow the backup rule from CLAUDE.md. Save affected rows to a JSON backup file in `tickets/{TICKET-ID}/data/backups/` with a timestamped filename.

#### Option B: CSV Hand-off

If the user prefers manual review or the BQ table is managed by another team, deliver the output CSV with a summary:
- Total stores geocoded
- Count by confidence tier (HIGH / MEDIUM / LOW / CENTROID / FAILED)
- List of QA-flagged rows

#### Option C: Google Sheet Update Instructions

If the data lives in a shared Google Sheet, provide cell-by-cell update instructions or a paste-ready two-column block (LATITUDE, LONGITUDE) ordered to match the sheet's row order.

## Fallback: Google Maps URL Extraction

If Nominatim fails for specific addresses (GEOCODE_FAILED), fall back to extracting coordinates from Google Maps:

1. Search Google Maps for the address
2. Extract lat/lon from the URL (format: `@lat,lon,zoom`)
3. Mark these with `GEO_MATCH_TYPE = "google_maps_manual"` and `GEO_CONFIDENCE = "MANUAL"`

This is a manual step. Do not automate scraping Google Maps (ToS violation). Present failed addresses to the user and offer to look them up one by one.
