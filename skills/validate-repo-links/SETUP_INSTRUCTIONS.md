# Setup Instructions: validate-repo-links

## Prerequisites

1. **Python 3.8 or higher**
   ```bash
   python3 --version
   ```

2. **PyYAML library**
   ```bash
   pip install pyyaml
   ```

   Or using pip3:
   ```bash
   pip3 install pyyaml
   ```

## Installation

The skill is already installed in your Claude Code environment at:
```
~/.claude-shared-config/skills/validate-repo-links/
```

## Initial Setup

### Step 1: Create Initial Index

Run the reindex command to scan your workspace and create the initial `.repo-index.yaml` file:

```bash
cd /Users/gabrielamyot/Developer/supervisr-ai/project-management
/validate-repo-links reindex
```

This will:
- Scan `app/micro-services/` for all `.repo-links.yaml` files
- Build the service interaction graph
- Create `.repo-index.yaml` in the project-management directory

### Step 2: Verify Installation

Run the validation command:

```bash
/validate-repo-links
```

Expected output:
```json
{
  "timestamp": "...",
  "total_repos": 5,
  "valid_repos": 5,
  "repos_with_errors": 0,
  "total_errors": 0,
  "overall_status": "valid"
}
```

### Step 3: Test Visualization

```bash
/validate-repo-links visualize
```

You should see a tree structure of your microservices and their relationships.

## Troubleshooting

### PyYAML not installed
```
ModuleNotFoundError: No module named 'yaml'
```

**Solution:**
```bash
pip3 install pyyaml
```

### Permission denied
```
PermissionError: [Errno 13] Permission denied
```

**Solution:**
Ensure the script is executable:
```bash
chmod +x ~/.claude-shared-config/skills/validate-repo-links/validate_repo_links.py
```

### No .repo-links.yaml files found

**Solution:**
1. Verify you're in the correct workspace
2. Check that `.repo-links.yaml` files exist in your service directories
3. The script looks in `app/micro-services/` by default

### Index file not found

**Solution:**
Run the reindex command first:
```bash
/validate-repo-links reindex
```

## Usage in Claude Code

Once set up, you can use the skill from anywhere in your workspace:

```bash
cd /Users/gabrielamyot/Developer/supervisr-ai/project-management

# From a service directory
cd /Users/gabrielamyot/Developer/supervisr-ai/app/micro-services/retell-service

# The skill always works relative to the workspace root
```

## Maintenance

### When to reindex

Run `/validate-repo-links reindex` when:
- Adding new microservices
- Changing service interaction patterns
- Modifying `.repo-links.yaml` files
- Moving repositories

### Regular validation

Run `/validate-repo-links` regularly to:
- Detect broken links after refactoring
- Verify infrastructure changes
- Ensure documentation accuracy

## Integration with Development Workflow

### Before major refactoring
```bash
/validate-repo-links
```
Ensures all links are valid before starting work.

### After moving/renaming
```bash
/validate-repo-links
```
Detects any broken links caused by the move.

### During code review
```bash
/validate-repo-links visualize
```
Shows service dependencies for reviewers.

### Periodic health checks
```bash
/validate-repo-links reindex && /validate-repo-links
```
Rebuilds index and validates everything.
